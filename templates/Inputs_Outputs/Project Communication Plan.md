# Project Communication Plan

## Purpose
Specifies regular communications during the project, including meetings, calls, status reports, and information flow.

## Inputs

| Item | Source |
|---|---|
| Stakeholder List | Project Charter |
| Project Schedule | Project Definition |
| Communication Protocols | Organizational Policy |

## Outputs

| Item | Description |
|---|---|
| Communication Matrix | Schedule of all communications |
| Meeting Agendas | Pre-defined agenda templates |
| Status Reports | Weekly/monthly progress reports |
| Communication Log | Record of all communications |

## Process

### 1. Identify Communication Needs
- **Who initiates**: Project Manager for status, Team Members for issues
- **Who receives**: Team, Sponsor, Stakeholders
- **Who chairs**: Project Manager (unless executive)
- **Who records**: Project Administrator or designated note-taker

### 2. Define Communication Types

| Type | Purpose | Frequency | Medium | Owner |
|---|---|---|---|---|
| Project Kickoff | Align expectations | Once | Meeting | PM |
| Daily Standup | Progress sync | Daily | Video/Phone | Team |
| Weekly Status | Progress report | Weekly | Email/Meeting | PM |
| Steering Committee | Decision making | Bi-weekly | Meeting | Sponsor |
| Risk Review | Issue escalation | As needed | Email/Meeting | PM |
| Change Request | Scope changes | As needed | Email | Team |

### 3. Communication Matrix

| Stakeholder | Information Type | Frequency | Channel |
|---|---|---|---|
| Sponsor | High-level status, risks | Weekly | Email + Meeting |
| Team | Task updates, blockers | Daily | Standup |
| Customers | Deliverable status | Bi-weekly | Email |
| Vendors | Milestone updates | As needed | Email/Call |

## Standing Agenda Format

### Weekly Status Meeting
1. **Review Previous Action Items** (5 min)
2. **Current Status** (10 min)
   - What's done
   - What's blocked
   - What's at risk
3. **Upcoming Milestones** (5 min)
4. **Issues & Decisions** (5 min)
5. **Action Items** (5 min)

## Communication Protocols

### Direct vs. Indirect Communication
- **Team to Team**: Direct communication allowed
- **Team to Sponsor**: Route through Project Manager
- **External Stakeholders**: All communication through PM

### Information Handling
- Use project management tool for all tracking
- Document decisions in meeting minutes
- Update project artifacts after each communication

## Assumptions

- Team has reliable internet/video conferencing
- Email is checked daily
- Meeting invites are accepted in calendar

## Dependencies

- Project management tool access
- Conference call setup
- Shared document repository

## Risks

- **Information overload**: Too many communications reduce effectiveness
- **Communication gaps**: Missed communications cause delays
- **Channel confusion**: Wrong medium used for urgency
