Should You Test?
Simply put, not all pages can (or should) be tested and anyone who says otherwise is doing you a severe disservice. Sometimes there are pages that just need to be fixed!
For example, if you were running Google PPC traffic to a landing page and were missing compliance elements - then you wouldn’t test the addition of these elements, you’d just add them!
By the way, if you aren’t sure what it means to have a compliant page, check out this Google PPC landing page resource below.
For a larger version of the above graphic, click here.
Before you start building up tests, make sure that you are actually testing a page that should be tested and testing elements that will make a difference!
You should start testing when you've arrived at a crossroad and there isn’t a clear way to proceed.
Here’s an example:
There is a page with a broken lead generation form - do you test it?
No! fix the form and move on.
Here’s an example of a local dentist who was wondering why his contact page wasn’t converting.
The form is...
awkward,
below the fold,
and has a ‘reset’ button.
Should he test? No! Make the necessary changes on this broken page and test something worthwhile!
Numbers are also crucial when deciding whether to test. If you don’t get enough traffic, you won’t be able to run a conclusive split test.
If you can’t get at least 100 converting actions on each variation, don’t bother running the test. If you do run the test, be very careful with the data because it is probably wrong.
You can still optimize your low traffic site and use other verification methods including:
Qualitative insights – Insights gathered from non-analytical sources such as heatmaps, user surveys, or personas.
Persona construction – A semi-fictional character depicting your ideal customer, in general you create several to account for different customer types.
Real time personalization – A type of user segmentation that dynamically inserts or provides specific content for that user segment.
Sequential Testing – A type of test comparing the results from two distinct dates. This method is often frowned upon, but is a good way for traffic challenged sites to get an idea about what’s working. Use sparingly if at all.
Split tests using micro conversions as indicators – Micro conversions are higher level conversions ,e.g., clicks, on higher funnel pages, e.g., homepage. They are used as indicators when you run a test that doesn’t have enough deep funnel conversion data. For example: if you are an ecommerce site with low traffic, you cannot run a test on your homepage and measure these changes at the sales level. When testing, you can look at the these higher level conversions as an indication of success.
