# Project Change Control

## Purpose
Process for managing changes to scope, schedule, and budget with clear approval workflow and audit trail.

## Benefits

- Fewer troubled projects
- Calculated adjustments to baseline with approval
- Coordinated work execution
- Audit trail for all changes
- Easy answers about scope/budget/schedule changes

## Consequences of No Change Control

- Unmet schedules
- Scope creep
- Unauthorized costs
- Increased project risks
- Unauthorized work
- Lower quality
- Liability exposure

## Change Control Process

### Step 1: Assess Change Request

**Purpose**: Quickly decide how to proceed

**Options**:
- Accept with no further analysis
- Request detailed impact analysis
- Defer or reject
- Close without analysis

**Assessment Criteria**:
- Impact on scope, schedule, budget
- Risk level
- Stakeholder importance
- Urgency

**Funding Agreement**:
- Estimate analysis cost
- Agree on funding source with change control board

### Step 2: Analyze Impact

**Purpose**: Provide detailed info for decision

**Analysis Steps**:
1. Reason for change
2. Priority of change
3. Scope impact (what's added/removed)
4. Proposed solution (options if applicable)
5. Cost impact (detailed)
6. Benefits (quantified)
7. Schedule change (days/weeks)
8. Resources required
9. Dependencies (other tasks affected)

**Minimization Techniques**:
- Exploit float in schedules
- Combine multiple changes into packages
- Deliver with planned releases
- Time changes to minimize disruption

### Step 3: Approve Change

**Purpose**: Make final decision

**Authority Levels**:
- Minor changes (<5% budget): Project Manager
- Moderate changes (5-15%): Change Control Board
- Major changes (>15%): Executive Sponsor

**Decision Factors**:
- Cost-benefit analysis
- Risk assessment
- Strategic alignment
- Resource availability

### Step 4: Implement Change Order

**Purpose**: Execute approved changes

**Tasks**:
1. Update project baseline
2. Revise project plans
3. Notify stakeholders
4. Update procedures
5. Monitor progress
6. Document completion

- **Date Submitted**: [Date]

[Detailed description of the requested change]

## Reason for Change
[Why this change is being requested]

## Impact Analysis
- **Scope Impact**: [What changes in deliverables]
- **Schedule Impact**: [Days/weeks affected]
- **Cost Impact**: [$ amount]
- **Resources**: [People, equipment, materials]
- **Dependencies**: [Other tasks affected]

## Proposed Solution

## Approval

- **Decision**: [Approved/Rejected/Deferred]

## Change Control Board (CCB)

| Role | Authority |
|---|---|
| Project Manager | Approve minor changes |
| Technical Lead | Approve technical changes |
| Finance Lead | Approve budget changes |
| Sponsor | Approve major changes |
