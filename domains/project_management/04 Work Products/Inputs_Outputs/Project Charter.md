# Project Charter

## Purpose
Formalizes the request from the sponsor detailing the goals, expectations and directions concerning the solution.

## Inputs

| Item | Source |
|---|---|
| Business Case | Strategic planning document |
| Project Scope | Requirements analysis |
| Stakeholder List | Project initiation |
| Sponsor Request | Initial proposal |

## Outputs

| Item | Description |
|---|---|
| Charter Document | Formal project authorization |
| Stakeholder Register | Identified roles and responsibilities |
| Project Objectives | Measurable goals and success criteria |
| Authority Statement | PM decision-making scope |

## Process: Building the Charter

1. **Gather Inputs**
   - Review business case and project scope
   - Identify sponsor and key stakeholders
   - Document initial goals and expectations

2. **Define Objectives**
   - Specific, measurable outcomes
   - Success criteria and acceptance conditions
   - Timeline and budget constraints

3. **Identify Stakeholders**
   - Sponsor and their authority level
   - Key team members and roles
   - Affected departments/groups

4. **Define Authority**
   - PM decision-making boundaries
   - Budget approval limits
   - Change control authority

5. **Document Constraints**
   - Budget limitations
   - Timeline requirements
   - Resource availability
   - Technical constraints

6. **Review and Approve**
   - Stakeholder sign-off
   - Sponsor approval
   - Archive for reference

## RACI Chart

| Activity | Project Manager | Sponsor | Team | Stakeholders |
|---|---|---|---|---|
| Define objectives | R | A | C | I |
| Identify stakeholders | R | A | C | I |
| Document constraints | R | A | C | I |
| Approve charter | - | A | C | I |
